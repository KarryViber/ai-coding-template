## AI 协作规则

### 工作流程

- 简要说思路就行，别过度思考 — 这是原型
- 写完快速 review，确保能跑
- 发现明显问题主动提醒

### 代码风格

- 注释尽量少
- 不要过度工程化 — 这是原型
- 不要碰我没让你改的代码

### Git

- 直接提交到 main
- 不要自动 commit — 先问我
- conventional commits + emoji

### 调试

- 修复前简要说根因
- 通过实际使用流程验证
- bug 重现时先查原始诊断

### 验证检查点

2. 测试失败场景
